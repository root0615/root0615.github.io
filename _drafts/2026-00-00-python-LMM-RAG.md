## 🔍 LLM(Large Language Model)이 무엇이냐?
> 풀이 뜻 대로 거대 언어 모델입니다.<br>
방대한 텍스트 데이터로 학습된 AI 모델로, 언어를 이해하고 생성하는 능력을 가집니다.<br>
GPT-4, Claude, Gemini 등이 있겠지요.<br>

❓ 어떻게 작동하는가?
- 수천억 개의 파라미터(가중치)로 구성된 신경망
- "다음에 올 단어/토큰을 예측"하는 방식으로 학습
- 학습 이후엔 지식이 고정됨 (학습 컷오프 존재)

📌 한계
- 학습 이후의 최신 정보를 모름
- 학습 데이터에 없는 특정 도메인 지식 부족
- 가끔 사실이 아닌 내용을 그럴싸하게 생성(할루시네이션)

## 🔍 RAG(Retrieval-Augmented Generation, 검색 증강 생성)이 무엇이냐?
> LLM의 한계를 보완하기 위해, 외부 데이터를 검색해서 LLM에게 제공하는 기법입니다.

학습 이후의 최신 정보를 모르니 그 이후 데이터로 제 입맛대로 학습시킬 수 있다는 것입니다.<br>
예를 들어, 우리 회사의 문서를 학습시켜 회사 내부의 전문가로 키워낼 수 있다는 것이지요.<br>
그래서 각 회사 사이트마다 챗봇이 나오나?? 그런것 같습니다.<br>

📝 작동 흐름
```
사용자 질문
    ↓
[검색] 벡터 DB 등에서 관련 문서 검색
    ↓
[증강] 검색된 문서 + 질문을 LLM에 전달
    ↓
[생성] LLM이 문서를 참고해서 답변 생성
```

❓ 왜 필요하지?
- 사내 문서, 최신 뉴스 등 LLM이 모르는 정보를 활용
- 답변의 근거를 제시할 수 있어 신뢰도 향상
- 모델을 재학습하지 않고도 지식 업데이트 가능

### 📄 LLM API 요청 코드 (빠른 시작 코드)
```python
# pip install groq  << 설치
from groq import Groq

# Groq 서버에 접속하기 위한 출입증 만들기. API 키는 내가 누구인지 증명하는 비밀번호
client = Groq(api_key="Groq 홈페이지에서 받은 API키 여기 넣으쇼")

"""
# client.chat.completions.create()
실제로 LLM 한테 메시지를 보내는 부분이다. 구조를 보면
client              # 위에서 만든 접속객체
    .chat           # 채팅 기능을 쓰겠다.
    .completions    # 텍스트 완성(응답생성)을 요청
    .create(...)    # 실제로 요청 생성
    
# messages=[{"role": "user", "content": "안녕!"}]
이게 핵심. LLM에게 대화 내용을 전달
role은 누가 말했느냐이고, content는 뭐라고 했는지

role에는 3가지가 존재
- system : AI 한테 역할/성격 부여("너는 데이터 분석가야")
- user : 내가 하는 말
- assistant : AI가 한말 (대화 히스토리 유지할 때 씀)
"""
response = client.chat.completions.create(
    # 어떤 AI 모델을 쓸지 지정. 70b는 700억개의 파라미터를 가진 모델. 클수록 복잡
    model="llama-3.3-70b-versatile",
    messages=[{"role": "user", "content": "안녕!"}]
)


"""
# AI 응답을 꺼내는 부분
response            # API가 돌려준 전체 응답 객체
    .choices        # 응답 후보들 (보통 1개)
    [0]             # 그 중 첫 번째
    .message        # 메시지 객체
    .content        # 실제 텍스트 내용
    
choices가 리스트인 이유 - AI가 여러 개의 응답 후보를 줄 수 있기 때문. 보통 [0]만 씀
"""
print(response.choices[0].message.content)
```
주석으로 상세한 설명을 나름 적어놨습니다.<br>
이 코드가 제대로 작동했다면 제가 `안녕!` 이라고 보낸 메세지에 LLM이 응답해준 내용이 콘솔에 뜰겁니다.

다음 단계로 `시스템 프롬프트`와 `대화 히스토리 구조`로 넘어가겠습니다.<br>

### 📄 LLM API 요청 코드 (시스템 프롬프트)
```python
"""
# messages 리스트에 system 역할이 추가된 것.
"role": "system", "content": "너는 데이터 분석 전문가야."

system은 AI한테 대화 시작전 역할을 부여한다.
같은 질문을 해도 system 프롬프트에 따라 답변 톤이나 깊이가 완전히 달라짐
"""
messages = [
    {
        "role": "system",
        "content": "너는 데이터 분석 전문가야. 이상탐지 결과를 쉽게 설명해주는 역할이야."
    },
    {
        "role": "user",
        "content": "Isolation Forest에서 anomaly score가 -0.35가 나왔어. 이게 뭘 의미해?"
    }
]

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=messages
)

print(response.choices[0].message.content)
```
> 달라진 부분은 바로 messages 리스트에 `system` 역할이 추가된 것입니다.<br>
`system`은 AI한테 대화 시작전에 역할을 부여하는 것이에요.<br>
같은 질문을 해도 system 프롬프트에 따라 답변 톤이나 깊이가 완전히 달라집니다.

### 📄 LLM API 요청 코드 (시스템 프롬프트 + 대화 히스토리 유지)
> LLM은 기본적으로 `기억이 없습니다`.<br>
매 요청마다 대화 내용을 통째로 넘겨줘야 이전 맥락을 기억합니다.<br>

```python
# 시스템 프롬프트 지정. 
messages = [
    {
        "role": "system",
        "content": "너는 데이터 분석 전문가야."
    }
]

while True:
    # 전달할 내용 input으로 받기
    user_input = input("나: ")
    if user_input == "exit":
        break

    # 전달할 내용(질문)을 히스토리에 추가
    messages.append({"role": "user", "content": user_input})
    
    response = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=messages  # 전체 히스토리를 매번 넘김
    )
    
    ai_reply = response.choices[0].message.content
    print(f"AI: {ai_reply}")

    # AI 답변도 히스토리에 추가(AI 자신이 어떻게 답변했는지도 알아야하니.)
    messages.append({"role": "assistant", "content": ai_reply}) # assistant로 추가

```

### 💻 LLM API 요청 코드 결과 (시스템 프롬프트 + 대화 히스토리 유지)
```
나: 안녕!
AI: 안녕하세요. 데이터 분석 전문가로서 다양한 도메인의 데이터를 처리하고 분석하는 데 전문가입니다. 데이터로부터 가치 있는 정보를 추출하고 의사 결정에 활용하는 데 도움을 드릴 수 있습니다. 데이터 분석과 관련해 어떤 도움이 필요하세요?

### 시스템 프롬프트를 적용하여 자신이 데이터 분석 전문가라는 걸 인지함 ###
-----------------------------------------------------------------
나: 혹시 점심 추천해줄래?
AI: 점심 추천해 드릴까요? 데이터 분석과 관련된 점심 추천은 다소 어렵겠지만, 일반적인 점심 추천을 해드릴 수 있어요. 오늘의 날씨와 기분에 따라 무엇을 드시면 좋을까요?

1. 한국식 점심: 비빔밥, 김치찌개, 비프스튜 등
2. 일식 점심:寿司, 라멘, 우동 등
3. 이탈리안 점심: 파스타, 피자, 브루스케타 등
4. 한식 프랜차이즈: 치킨, 피자, 샌드위치 등

무엇이 마음에 드시나요? 혹은 다른 선호도에 따라 추천해 드릴까요?
-----------------------------------------------------------------
나: 음.. 2번
AI: 일식 점심을 드시고 싶으신 것 같네요. 일식은 다양한 종류가 있습니다. 여기 몇 가지 추천해 드릴게요:

1. **寿司**: 생선과 밥이 함께 있는 일본의 대표 음식입니다. 초밥이 좋다면, 롤이나 네기리 등 다양한 종류의寿司를 드실 수 있습니다.
2. **라멘**: 일본의 대표 국물면입니다.豚骨ラーメン, 鶏白湯ラーメン, 미소 라멘 등 다양한 맛을 즐길 수 있습니다.
3. **우동**: 두꺼운 국수와 함께 오이, 당근, 가츠오부시 등이 들어간 일본의 국수 음식입니다.
4. **덮밥**: 일본식 덮밥으로, 밥 위에 재료를 올려 먹는 음식입니다. 인기 있는 맛은 가라아게 덮밥, 가유 덮밥 등이 있습니다.

어떤 메뉴가 가장 마음에 드시나요? 혹은 특정 재료를 선호하시나요?
-----------------------------------------------------------------
나: 아까 리스트 중에 1번이 뭐였지??
AI: 아까 점심 추천 리스트에서 1번은 **한국식 점심**이었습니다. 비빔밥, 김치찌개, 비프스튜 등 한국의 전통 음식을 포함했습니다. 재방문하기 좋을까요?

```
📌 핵심 구조 시각화<br>
```
messages 리스트 (매 요청마다 통째로 전송)

이렇게 챗봇의 기본 뼈대가 완성되었습니다.
```
