## 🔍 Isolation Forest
> 이상치는 정상 데이터보다 훨씬 빠르게 `고립`됩니다.

이게 빠르게 고립된다는게 처음 접할 때는 무슨소리야? 라고 생각했던 기억이 있습니다.<br>
이해하기 쉽게 생각했을 때, 정상데이터와 이상데이터가 값이 정말 많이 차이 난다면 아마 정상데이터는 모여있고 이상데이터는 저 멀리 떨어져 있을 겁니다.<br>
그러면 분리하기 위해 랜덤하게 여러번 선을 그었을 때 정상 데이터보다 이상데이터들이 빠르게 분리가 된다는 겁니다.<br>
그림을 보고 원리를 설명하겠습니다.
![IF](/assets/img/ML_DL/isolation_forest.png)<br>
구분하기 위해 3번의 선을 랜덤하게 그었을 경우 현재 고립이 되는건 빨간색의 2개의 데이터가 고립이 됩니다. <br>
다른 데이터들은 아직 고립이 되지 않았고 좀더 랜덤하게 선을 긋다 보면 고립이 되겠지요.<br>
이처럼 몇번 긋지도 않았는데 고립이 되어버린 데이터는 이상치일 확률이 높다고 판단합니다.<br>
그래서 각 데이터에 대하여 이상치 점수를 계산하고 점수가 높을수록 이상치라고 판단합니다.<br>

### 📌 원리
무작위로 데이터를 분할(트리 생성)할 때, 정상 데이터는 많은 분할이 필요하지만 이상치는 소수의 분할만으로 금방 고립됩니다.<br>
고립까지 필요한 분할 횟수가 곧 이상도(anomaly score)입니다.<br>

### 📌 사용 이유
- 레이블이 없는 대용량 데이터에 빠르고 효과적입니다.
- 학습 속도가 빠릅니다.
- 고차원 데이터에 강합니다.

## 🔍 OCSVM (One-Class SVM)
> 정상 데이터만 학습해 그 경계 밖을 이상치로 판단합니다.

OCSVM은 하나의 클래스로 이루어진 데이터에서 정상 데이터와 이상치를 나누는 평면을 찾는 비지도 학습 알고리즘입니다.<br>
isolation forest가 이상한 놈은 빨리 혼자 남는거라면, OCSVM은 정상 데이터 주위에 울타리를 치고 울타리 밖에 나간 놈들을 이상치로 본다고 생각하면 됩니다.<br>
![OCSVM-1](/assets/img/ML_DL/OCSVM-1.png)<br>
원점은 `아무 데이터도 없는 빈 공간`을 대표합니다.<br>
OCSVM은 정상 데이터 덩어리와 원점 사이에 최대한 멀리 경계선을 그어 둘을 분리합니다.<br>
현재 빨간색 이상 데이터는 계산하지 않고 진행합니다.<br>
만약 데이터가 단순하게 뭉쳐 있다면 이렇게 직선으로 바로 되겠지만 현실은 그렇지 않습니다.<br>
고차원일 경우에는 직선으로는 구분지을 수 없으니까요.<br>
그래서 커널 트릭이라는 것이 있습니다.<br>
![OCSVM-2](/assets/img/ML_DL/OCSVM-2.png)<br>
[출처] 접니다. 그림작가는 Claude<br>

그러니까 고차원의 형태를 커널트릭을 사용하면 2차원 평면에 있는 것처럼 변환을 해줄 수 있습니다.<br>
그러면 경계선을 긋기 쉬워지겠지요.<br>
이때 그어진 경계선이 정상과 이상을 구분하는 기준이 됩니다.<br>
경계 안쪽이면 정상, 바깥쪽이면 이상이 되는 것이죠.<br>
원리를 다시 정리해서 말해보겠습니다.<br>

### 📌 원리
정상 데이터 주변에 초평면(경계)을 학습합니다.<br>
커널 함수로 고차원 공간에 매핑한 뒤, 원점으로부터 정상 데이터를 최대한 분리하는 경계를 찾고, 이 경계밖의 점을 이상치로 판별합니다.<br>

### 📌 사용 이유
- 정상 데이터만 존재하는 상황에서 유효
- 커널 선택으로 비선형 경계를 표현할 수 있어 복잡한 분포에도 적용 가능

## 🔍 Autoencoder
> 입력을 압축했다 복원할 때, 이상치는 복원 오류가 큽니다.

비유를 해서 말하자면,<br>
우리가 사용하는 압축 프로그램으로 파일이나 폴더를 압축하면 파일 크기가 줄어들고 여러 파일들이 하나의 압축 파일로 변환되지 않습니까?<br>
그리고 압축 풀면 압축하기 전에 내용 그대로 나옵니다.<br>
정상 데이터를 압축하고 복원하는 방법을 학습한 모델이 F라 하면, 다른 정상 데이터도 F 방법을 사용해서 압축, 복원하면 완전 동일하진 않지만 차이는 거의 보이지 않을 겁니다.<br>
하지만 이상한 데이터를 F 방법으로 압축하고 복원하면 분명 완전히 다른 결과가 나올거에요.<br>
이러한 원리를 이용했다고 생각하시면 됩니다.<br>

![AE](/assets/img/ML_DL/AE.png)<br>
[출처]<https://yeong-jin-data-blog.tistory.com/entry/Autoencoder><br>

### 📌 원리
인코더로 입력을 저차원 잠재 벡터로 압축하고, 디코더로 다시 복원합니다.<br>
정상 데이터 위주로 학습하면 정상은 잘 복원되지만, 이상치는 복원이 어려워<br>
재구성 오류(reconstruction error)가 커집니다.<br>

### 📌 사용 이유
- 비지도 이상 탐지의 핵심 모델
- 차원 축소 목적으로 사용
- 다양한 도메인에 적용 가능
- 잠재 공간을 통해 데이터의 본질적인 패턴 학습

## 🔍 LSTM Autoencoder
> 시간 순서를 기억하는 LSTM으로 시계열 패턴을 압축, 복원합니다.

LSTM은 말 그대로 LSTM와 Autoencoder 두 개념의 합체입니다.<br>
LSTM이 뭔지 간단하게 설명하면,<br>
LSTM은 시간 순서가 있는 데이터를 읽으면서 `중요한 것은 기억하고, 필요없는 건 잊는` 셀입니다.<br>
일반 신경망과 달리 과거 정보를 들고 다닙니다.<br>
이 LSTM을 인코더 자리에 넣으면, 시계열 전체를 순서대로 읽어서 마지막 잠재 벡터 하나로 압축합니다.<br>
디코더 자리 LSTM은 해당 잠재 벡터를 역으로 복원합니다. <br>
기존 Autoencoder와 동일하게 정상 패턴만 학습하여 재구성 오류 차이를 보고 이상치를 판별합니다.<br>

![LSTM AE Flow](/assets/img/ML_DL/LSTM_AE_Flow.png)<br>
[출처]<https://skyeong.net/316>

### 📌 원리
Autoencoder의 인코더, 디코더를 LSTM 셀로 대체합니다.<br>
LSTM의 셀 상태(cell state)와 은닉 상태가 시간에 따른 패턴(장기 의존성)을 기억하며, 시계열 전체를 하나의 잠재 벡터로 압축 후 복원합니다.<br>

### 📌 사용 이유
- 센서로그, 주가, 의료 신호 등 순서가 중요한 시계열 이상 탐지에 특화
- 시간 패턴 이탈을 잡아냄

## 🔍 CNN Autoencoder
> 합성곱으로 공간적 패턴을 추출해 압축, 복원하며 이상을 탐지한다.<br>

CNN 오토인코더 구조는 LSTM AE랑 같습니다.<br>
다른 건 딱 하나, 인코더와 디코더 자리에 LSTM 대신 CNN을 쓴다는 것입니다.<br>
일단 CNN이 뭔지 잡고 가자면,<br>
3 x 3 모양의 필터(돋보기)를 이미지 위에 쭉 밀면서 "이 부분에 특징이 있나??"훑습니다.<br>
전체를 보지 않고 이곳 저곳을 저 돋보기로 훑는 방식입니다. <br>
이러한 방법으로 특징을 추출해냅니다.<br>
그리고 Pooling(절반으로 압축)을 반복합니다. <br>
64x64 --> 32x32, --> 16x16 --> 잠재벡터.<br>
정상 이미지의 핵심 패턴만 남깁니다.<br>
Pooling은 대표값만 남기고 나머지를 버리는 역할을 한다고 생각하시면 됩니다.<br>

디코더는 upsample로 Pooling의 반대로 진행되며 Upsample과 Conv를 반복해서 <br>잠재벡터를 다시 원래 크기로 복원합니다.<br>

### 📌 원리
합성곱(Conv) 레이어가 이미지나 1D 신호의 지역적 패턴(엣지, 텍스처 등)을 필터로 추출하고, 풀링으로 압축합니다.<br>
디코더는 역합성곱(Transposed Conv)으로 복원하며, 복원 오류가 큰 부분이 이상 영역입니다.<br>
레이어의 조종류만 다른뿐 원리는 빗아여 배우느지 지장이 없실 거라 생각됩니다.

### 📌 사용 이유
- 이미지 불량 탐지, 의료영상, 스펙트로그램 등 공간 패턴이 중요한 데이터에 적합.<br>
위치까지 파악 가능
